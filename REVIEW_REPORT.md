## 二、阻塞级问题 🚨

### B1. README 与 DESIGN 对 MAX98357 SD_MODE 接法直接矛盾

**问题**：
- README 第 43 行写 `SD_MODE (接GND = 左声道 + 1/2增益)`
- DESIGN §3.4.1 写 `SD_MODE 接 VDD → L+R 混合为单声道输出`
- PRD 变更日志也标注"SD_MODE 改 VDD"
- **影响**：按 README 操作会得到左声道单声道输出，而非 L+R 混合，硬件接线错误

**核实**：✅ 真实存在，且比原报告描述更严重——两份文档直接矛盾

**修复**：✅ 已修复 — README SD_MODE 已改为 VDD，PRD §7.2 已补充 MAX98357 配置说明

### I1. NVS 命名空间跨文档冲突

**问题**：
- README 第 227 行写 `"audiobook"` 命名空间
- config.h 第 83 行和 DESIGN §5.5.1 均为 `"tapebook"` 命名空间
- 若按 README 示例代码编写，将无法读取已有断点数据

**核实**：✅ 真实，原报告遗漏此问题（原 I2 仅关注 key 前缀不一致，实际更严重的命名空间冲突未发现）

**修复**：✅ 已修复 — README NVS 命名空间已改为 `"tapebook"`

---

### I2. 跨文档 NVS Key 命名风格差异（原报告重点修正）

| 文档 | Key 风格 | 实际一致性 |
|------|---------|-----------|
| PRD | `book_N_name`, `book_N_pos`, `last_track_idx` | 语义与 DESIGN 一致 |
| DESIGN | `book_0_name`, `volume`, `play_mode`, `ab_repeat` | ✅ |
| DETAILED_DESIGN / config.h | `last_track_idx`(宏), `last_position`, `volume` | ✅ 与 DESIGN 完全一致 |

> **原报告错误**：一致性矩阵标注 DETAILED_DESIGN 用 `settings_*` 前缀 — **实际无此前缀**，key 与 DESIGN 完全一致。

**核实**：⚠️ 基本不成立 — PRD 的 `book_N_name` 和 DESIGN 的 `book_{idx}_name` 是同一模式的不同写法，语义一致。真正的问题是命名空间冲突（见 I1），而非 key 前缀。

**建议**：修正一致性矩阵，以 config.h 宏定义为唯一基准。

### I5. 96kHz I2S 切换杂音

DESIGN §5.2.3 表格明确列出 96000Hz 上限。`i2s_stream_set_clk` 重初始化 I2S 驱动是已知风险。

**核实**：✅ 真实，需实测验证。

**建议**：实测验证，必要时限制最高 2.5x（44100→66250）。

**核实**：✅ 真实。

### I8. ESP-IDF/ADF 版本推荐不一致

**问题**：
- 原评审报告技术风险表推荐"锁定 IDF v5.1 + ADF v2.5"
- DESIGN §10.2 和 README 均指定 **v5.3 + v2.7**

**核实**：✅ 真实 — 原报告版本推荐与项目文档矛盾。项目文档（DESIGN/README）统一为 v5.3 + v2.7，这是正确的 LTS 稳定版组合。

**修复**：✅ 已修复 — 评审报告技术风险表已修正为 v5.3 + v2.7

---

### I9. 30 秒 NVS commit 频率较高

NVS 寿命约 10 万次擦写，30s/次 = 2880 次/天，能用 34 年。但单次 commit 10ms 级可能卡顿。

**核实**：✅ 真实。2880 次/天远低于寿命上限，但卡顿风险需实测。

**建议**：先实测耗时，必要时改为 60s + 仅状态变化时 commit。
