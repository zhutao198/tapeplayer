### R002 教训（T-001）
- **现象**：第一次 `tar czf D:/u8g2_backup_20260703.tar.gz components/u8g2/` 失败（`Cannot connect to D: resolve failed`），但 `rm -rf` 仍执行
- **正确用法**（Git Bash on Windows）：
  ```bash
  # 错误：tar czf D:/backup.tar.gz src/    ← D:/ 被当作 host 解析失败
  # 正确：tar czf /d/backup.tar.gz src/    ← POSIX 路径 /d/ = D:\
  # 或：MSYS_NO_PATHCONV=1 tar czf D:/backup.tar.gz src/
  ```
- **教训**：删前必须**先验证 tar 成功**（`tar tzf` 抽查），再 `rm -rf`。两命令分开跑，不要 `&&` 链式
- **未来**：所有"先备份后删"操作必须分两步：① tar + `tar tzf` 验证 ② 再 rm
