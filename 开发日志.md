### R003 教训（T-002 Git Bash on Windows + ESP-IDF）
- **现象 A**：Git Bash 下 `./build.bat` 不被 bash 解释（只 echo 不跑 call），`cmd /c` 开新窗口，`cmd //c` 路径转换失败
- **正确用法**：`powershell -NoProfile -ExecutionPolicy Bypass -Command "Remove-Item Env:MSYSTEM -ErrorAction SilentlyContinue; & 'D:\esp\v5.5.3\esp-idf\export.ps1' | Out-Null; \$env:ADF_PATH='D:\esp\esp-adf'; idf.py build"`
- **教训**：
  1. **必须**在 PowerShell 内部 `Remove-Item Env:MSYSTEM`（bash 子 shell `unset` 不传给 PowerShell）
  2. `export.ps1` 通过 `&` dot-source 调，保留环境变量
  3. **不要**用 `cmd /c "build.bat"` 跑 build.bat（开新窗口），用 PowerShell 调 `idf.py build`
- **未来**：所有 ESP-IDF / ESP-ADF build 必走 PowerShell + MSYSTEM unset 模式

### R003 教训（T-003 ESP-IDF 子模块 fix 模式）
- **现象**：ESP-IDF 5.5.3 install 时 micro-ecc / bt 控制器子模块只 clone 元数据，**没 fetch objects** → HEAD bad object
- **修复模式**（每个子模块 3 步）：
  1. `cd <submodule_path>` （如 `D:/esp/v5.5.3/esp-idf/components/bt/controller/lib_esp32`）
  2. `git fetch origin`（拉所有 branches/tags）
  3. `git reset --hard <ESP-IDF 锁定的 commit>`（从 `.gitmodules` sbom-hash 或 `HEAD` 文件读）
- **批量化建议**：下次 install ESP-IDF 后跑个 batch 脚本一次性 fix 所有坏子模块
- **未来**：建立 `tools/fix_idf_submodules.sh` 一键修复

### R003 教训（T-004 ADF 5.5 组件引用方式变化）
- **现象**：ESP-ADF 5.5 的 `audio_pipeline` 等组件已迁移到 IDF component manager
- **错误做法**：`main/CMakeLists.txt REQUIRES audio_pipeline`（被弃用）
- **正确做法**：`main/idf_component.yml dependencies: espressif/audio_pipeline: "*"`（用 idf component 拉）
- **教训**：遇到 "The component X could not be found" + 提示"moved to IDF component manager" 时，**改用 idf_component.yml 引用**
